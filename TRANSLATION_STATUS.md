## Translation Standards

### Requirements for All Translations

1. **Terminology Consistency**
   - Use terms from trilingual glossary
   - Keep technical acronyms in Latin characters (WCAG, ISO, UX, UI)
   - Maintain requirement IDs in English (S1, A1, G1, E1)

2. **Arabic RTL Requirements**
   - Include RTL display note at top of file
   - Test rendering in RTL-capable viewers
   - Keep code examples and metrics in LTR

3. **Cultural Adaptation**
   - Adapt examples to local context where appropriate
   - Maintain technical accuracy across all languages
   - Use culturally appropriate metaphors and references

4. **Quality Assurance**
   - Native speaker review required
   - Technical accuracy verification
   - Consistency check against glossary

### Translation Process

1. **Claim a document** - Open an issue indicating which file you want to translate
2. **Use the glossary** - Reference glossary/{language}.md for consistent terminology
3. **Follow the pattern** - Use existing translated files as templates
4. **Submit PR** - Include both the translation and any glossary additions
5. **Review process** - Core team + community review for accuracy
